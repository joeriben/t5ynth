T5ynth

A text-to-audio synthesizer that navigates the T5 embedding space of a diffusion audio model.
This guide covers how to use T5ynth and the ideas behind it.

T5ynth is a JUCE-based synthesizer plugin (Standalone / VST3 / AU) that deliberately repurposes Stability AI's Stable Audio Open model as an oscillator — not to generate finished music, but to produce raw sonic material for human–nonhuman artistic collaboration. Where text-to-audio models are designed to substitute creative labor (type a description, get a result), T5ynth inverts this relationship: the AI produces material, the human creates. This is not a consumer tool — it is an instrument that requires musicianship.

T5ynth is a personal side-project by Prof. Dr. Benjamin Jörissen, UNESCO Chair in Digital Culture and Arts in Education (UCDCAE), Friedrich-Alexander-Universität Erlangen-Nürnberg, and part of the UCDCAE AI Lab Software Collection.

T5ynth is inspired by two research projects:

  • AI for Arts Education (AI4ArtsEd), conducted together with the University of Cologne and the German Research Institute for Artificial Intelligence (DFKI) Kaiserslautern
  • ComeArts Across, a research project for the development of digital cultural teacher education

Both funded by the Federal Ministry for Education, Family, Senior Citizens, Women and Youth (BMBFSFJ).

T5ynth is dedicated to my dear colleague at the DFKI, musician and AI researcher Dr. Stephan Baumann, without whom AI4ArtsEd would not have come into being.

Setup: Models & Python Backend

T5ynth consists of two parts: a JUCE audio plugin (the synthesizer UI and DSP) and a Python backend (the inference engine that runs the diffusion model). Both must be set up before T5ynth can generate audio.

Python Environment

T5ynth needs a Python virtual environment with the required ML libraries (PyTorch, Diffusers, Transformers, etc.). From the repository root:

python3 -m venv .venv source .venv/bin/activate pip install -r backend/requirements.txt

T5ynth automatically finds the venv at .venv/ relative to the application. No manual path configuration needed.

Model Download

T5ynth requires at least one diffusion model. Models are not bundled — they must be downloaded separately (Stable Audio Open 1.0 is ~11 GB).

Via the Settings panel (recommended)

  1. Launch T5ynth and click Settings (gear icon in the status bar)
  2. Select the model you want to install:
    • Stable Audio Open 1.0 — requires a HuggingFace token (free account). Enter the token, then click Download.
    • Stable Audio Small — downloads from GitHub Releases, no token needed.
  3. Wait for the download to complete (progress bar shown). The model is stored in the application data directory.

Manual download

Alternatively, download models via the HuggingFace CLI or browser and place them in the models directory:

PlatformModels Directory
macOS~/Library/T5ynth/models/
Linux~/.local/share/T5ynth/models/

Each model lives in its own subdirectory (e.g., stable-audio-open-1.0/). T5ynth auto-detects models by looking for model_index.json (Diffusers format) or model_config.json (native format).

Device Selection

T5ynth auto-detects the best available compute device:

  • Apple Silicon Mac → MPS (Metal Performance Shaders), ~3× faster than CPU
  • NVIDIA GPU (Linux) → CUDA
  • No GPU → CPU fallback (works, but slow)

You can override the device in the generation section. The device toggle switches between GPU and CPU.

First Generation

On first launch, the status bar shows backend connection status. Once you see "Ready", type a prompt and click Generate. The first generation takes longer because the model is loaded into GPU memory on demand. Subsequent generations are faster.

If the status bar shows an error, check: (1) Is the Python venv set up? (2) Is at least one model downloaded? (3) Is there enough disk space and RAM/VRAM? Stable Audio Open 1.0 needs ~4 GB VRAM.
Part I

1. The Generation Section (Oscillator)

T5ynth does not have a traditional oscillator. Instead, it repurposes a text-to-audio AI model as a sound source — not to generate finished music on command, but to produce raw sonic material that you shape as a musician. The diffusion model (Stable Audio) becomes an oscillator whose output is steered by navigating a 768-dimensional embedding space. This is a deliberate inversion: the AI is not the creator, you are. The model provides material; you make music from it.

Prompts A & B

You have two text input fields. Each prompt is encoded into a 768-dimensional embedding vector by a T5 language model. To understand what that means: an embedding is a way of representing the meaning of a text as a point in a high-dimensional space. Words and phrases that are semantically related end up close together; unrelated concepts end up far apart. Every dimension captures some aspect of meaning — not a single human-readable feature, but a learned statistical pattern. (For an excellent visual introduction, see Jay Alammar's Illustrated Word2Vec.)

These embedding vectors — not the text itself — are what steers the diffusion model. T5ynth operates on them as geometric objects: moving, blending, scaling, and perturbing them before any audio is generated.

  • Single prompt — enter text in Prompt A only. The model generates audio matching that description.
  • Two prompts — enter text in both A and B. The Alpha slider navigates between them in embedding space.
Prompts should describe sound qualities, not musical genres. "A warm resonant drone with metallic harmonics" works better than "ambient techno." The model has no reliable concept of genre.

Generation Controls

ControlRangeWhat It Does
Alpha -2.0 … +2.0 Navigates between Prompt A (−1) and Prompt B (+1) in embedding space, not in audio. This is not a crossfade or mix of two audio signals — it is movement through a semantic space. The midpoint (0) is not "half A plus half B" in any audible sense; it is a new point in meaning-space that the model interprets independently. For example, if Prompt A is "bowed cello, sustained" and Prompt B is "crackling fire", the midpoint will not sound like cello mixed with fire — it might produce something like a warm, textured, resonant hum that resembles neither source directly. Values beyond ±1 extrapolate — pushing past either pole into regions of the embedding space that no text prompt would naturally reach. The slider uses a quadratic curve for fine control near the center.
Magnitude 0.001 … 5.0 Scales the embedding vector's length. At 1.0 the prompt influence is normal. Lower values drift toward the model's "average" sound; higher values push the model harder (can get unstable above ~2.0).
Noise 0.0 … 1.0 This is not audio noise — it is semantic noise: random perturbation in meaning-space, a dose of chaos applied to the embedding before the model ever generates a sound. Even small values (0.05–0.15) introduce subtle variation between otherwise identical generations. Higher values scramble the prompt — useful for exploration, less for precision.
Duration 0.1 … 47 s Length of generated audio. Short values (0.5–3s) work well for loops and wavetable sources. Longer values create evolving textures.
Steps 1 … 100 Diffusion denoising iterations. More steps = higher quality but slower. The useful range depends on the model: Stable Audio Open 1.0 benefits from 20–100 steps (20 is a good default, higher values refine detail). Stable Audio Small converges quickly — above 8 steps there is no positive effect.
CFG Scale 1.0 … 15.0 Classifier-Free Guidance — how strictly the model follows your prompt. Model-dependent: Stable Audio Open 1.0 works well at 5–8 (default 7). Stable Audio Small should stay at 1 — higher values produce artifacts rather than better adherence. In general: low values are loose and dreamy, high values are literal but can sound harsh.
Start Position 0.0 … 1.0 Inpainting offset within the diffusion timeline. 0.0 gives you the attack/onset of a sound. ~0.5–0.7 gives you the sustained body, skipping transients — if there were any in the generated sample.
Seed -1 … 999 999 999 Controls random state. Same seed + same settings = identical output (cross-platform). Use the Random button to automatically pick a new seed on each generation.
HF Boost on / off Compensates for high-frequency rolloff inherent to the VAE decoder. Whether you prefer the brighter boosted or the darker raw sound is a matter of taste.
Auto on / off Automatic re-generation after every parameter change in the oscillator section. (Planned feature — not yet implemented.)
This is not a static oscillator. The generation parameters above — Alpha, Axes, Magnitude, Noise — can all be modulated by the Drift & Regenerate system. With Drift active, these values shift continuously, and T5ynth can automatically re-generate audio as they move. The T5 oscillator becomes a living sound source that evolves over time, not a one-shot sample generator. See section 6 for details.

Model & Device

Three model slots are available: Stable Audio Open 1.0, Stable Audio Small, and AudioLDM2 (planned). The device toggle switches between GPU (MPS on Mac, CUDA on Linux) and CPU. GPU is typically 3× faster on Apple Silicon.

3. Dimension Explorer

The Dimension Explorer gives you direct access to all 768 individual dimensions of the T5 embedding space. While prompts and semantic axes offer human-readable ways to navigate this space, the DimExplorer lets you see and manipulate the raw numbers.

What You See

After generating audio, T5ynth receives the 768-dimensional embedding vectors for your prompts. The DimExplorer displays these as a row of vertical bars — one per dimension, sorted by magnitude (the most significant dimensions appear on the left).

  • With two prompts: Each bar shows the difference between Prompt A and Prompt B for that dimension. Green bars point toward A, orange bars toward B. The leftmost bars are the dimensions where your two prompts differ most — the dimensions that define the sonic difference between A and B.
  • With one prompt: Bars show the absolute activation values of the embedding.

What You Can Do

Drag any bar vertically to apply a per-dimension offset. Edited bars turn blue. Click "Apply + Generate" to send these offsets to the inference engine, where they are added directly to the embedding before diffusion runs.

What This Means

This is the most granular level of control T5ynth offers. It bypasses language entirely — you are directly sculpting the 768-dimensional vector that guides the diffusion model. What each dimension "does" sonically is largely opaque: the T5 embedding space was trained for language tasks, not audio, and its internal structure does not map to perceptual audio categories in any documented way. Small changes may do nothing, or they may radically alter the output. This is a research tool for probing the space, not a precision instrument.

A compact mini-view of the DimExplorer is always visible below the Axes panel. Click it to open the full interactive overlay. The overlay provides Undo, Redo, and Reset All controls for your edits.

4. Engine Modes: Sampler & Wavetable

T5ynth can play back generated audio in two modes:

Sampler Mode

Plays the generated audio buffer directly. Pitch follows MIDI via time-stretching (Signalsmith Stretch). This mode preserves the full character of the generation.

  • Loop Mode — One-shot, Loop, or Ping-Pong
  • Crossfade — 0–500 ms equal-power crossfade at the loop boundary
  • Loop Optimize — Cross-correlation algorithm that finds the smoothest loop point (Low or High precision)
  • Normalize — Peak-normalizes the play region

Wavetable Mode

Extracts pitch-synchronous frames from the audio to build a scannable wavetable. The Scan slider morphs between frames, and pitch follows MIDI directly (band-limited harmonic synthesis).

  • Scan (0.0–1.0) — Selects the current frame position in the wavetable. Can be modulated by LFOs, envelopes, or drift.

Voice Count

Applies to both modes: Mono, 4, 6, 8, 12, or 16 voices. Gain auto-scales (equal power: each voice at 1/√N). Oldest notes are stolen when the limit is reached.

6. Drift & Regenerate

Drift & Regenerate is what makes T5ynth more than a sample player or wavetable synth.
Without it, you generate a sound and then shape it with conventional synth tools — the source stays fixed. With Drift & Regenerate, the sound source itself continuously evolves. T5ynth becomes an instrument that transforms in real time, autonomously navigating the embedding space while you play. The oscillator is no longer a static waveform — it is a living, moving trajectory through a 768-dimensional sonic landscape.

The Asynchronous Feedback Loop

Diffusion inference is never real-time — a single generation takes seconds, not milliseconds. But T5ynth creates an asynchronous feedback loop between the modulation system and the T5 oscillator that keeps you in the flow:

While the synth is playing, multiple drift LFOs continuously and independently shift the Alpha slider and the semantic axes. Each LFO moves at its own rate and depth — Alpha might slowly sweep from Prompt A toward B over 30 seconds, while Axis 1 (e.g., tonal/noisy) oscillates on a different cycle, and Axis 2 (e.g., sustained/rhythmic) drifts at yet another pace. These modulations run simultaneously and asynchronously, tracing a complex, never-repeating path through the embedding space.

When Auto or 1st Bar regen mode is active, T5ynth monitors how far the modulated embedding has drifted from the last generated state. Once the drift exceeds a threshold, it fires a new inference request in the background. The current sound keeps playing uninterrupted. When the new generation arrives — typically a few seconds later — it is crossfaded into the playback buffer, smoothly replacing the old sound with the new one.

The result is a continuous stream of new generations, each one produced from a slightly different position in the embedding space between and around the two prompts. The sound never loops back to exactly where it was. Every few seconds, a new variation emerges — not randomly, but along the trajectories defined by your drift LFO settings. You hear a sound that perpetually transforms: sometimes closer to Prompt A, sometimes to B, sometimes in between, colored by whichever semantic axes are drifting, at whatever pace you have set.

Meanwhile, the conventional synth chain (filter, envelopes, LFOs, effects, sequencer) continues to process the output in real time. You are simultaneously shaping the sound and the process that creates the sound.

The Drift section provides three slow LFOs (0.001–2.0 Hz) designed for organic, evolving textures. Unlike the main LFOs, drift can target generation-level parameters — Alpha and the three semantic axes. This is what makes drift unique: it doesn't just modulate how the sound is played back, it modulates what sound is generated.

10. Arpeggiator

The arpeggiator sits after both the step sequencer and the generative sequencer — the chain is always: Sequencer → Arp → Synth Engine. It processes whatever note input it receives, whether from manual steps, GEN mode, or live MIDI. With the generative sequencer, this means an already evolving pattern gets further expanded by the arpeggiator — the combination can produce complex, self-developing textures.

ControlOptionsNotes

11. Presets

T5ynth uses .t5p preset files — a binary container that stores:

  • All parameter values (100+ parameters)
  • Prompt text (A and B)
  • The generated audio itself (PCM, so the preset sounds right without re-generating)
  • Embedding vectors (768 floats each, for DimExplorer context)
  • Seed, device, model selection

Use the Import / Export buttons in the preset panel to load and save.

12. Strategies for Finding Sweet Spots

T5ynth is not a sweet-spot synth. Unlike subtractive synths where most parameter combinations produce usable sound, T5ynth's generation process is exploratory and unpredictable. Many combinations produce nothing interesting. That's by design — the instrument rewards patient exploration, not preset-surfing.

1. Start with the source

Get a generation you like before touching synth parameters. A great filter envelope cannot save a bad generation. Experiment with prompts, seeds, and short durations (1–3s) to quickly iterate.

2. Use low step counts for speed

While exploring, keep Steps low to iterate quickly. With Stable Audio Small, 8 steps is already the ceiling — more steps add nothing. With Stable Audio Open 1.0, start at 20 and only go higher once you've found a direction worth refining.

3. Exploit the seed

Same prompt + different seed = very different audio. If a prompt feels right but the result doesn't, change the seed before changing the words. Generate 5–10 seeds before concluding a prompt doesn't work.

4. Alpha for morphing, not mixing

Alpha is not a volume crossfade. It interpolates in embedding space — the midpoint (0.0) is often not a blend of A and B but something new. Use Alpha to discover unexpected sounds between your two prompts. Push it past ±1.0 to extrapolate.

5. One axis at a time

Semantic axes are most effective used individually. Stacking three axes at extreme values dilutes each one. Start with one axis and find its useful range before adding a second.

6. Wavetable mode for pitched material

If you need stable pitch tracking (melodic lines, chords), switch to Wavetable mode. Sampler mode preserves the original character but pitch-stretching can introduce artifacts at extreme intervals.

7. Short loops + Scan modulation

Generate 1–2s of audio, switch to Wavetable, and modulate Scan with a slow LFO or Env1. This creates evolving timbres from a single generation — the classic wavetable synthesis approach, but with AI-generated waves.

8. Drift for evolution, not randomness

Set Drift LFO rates very low (0.01–0.05 Hz) and depths moderate (0.2–0.5). Target Alpha or an Axis. Enable Auto-regen. This activates the asynchronous feedback loop — the synth slowly transforms itself, producing a continuous stream of new generations. Far more interesting than random Noise injection.

9. Gen Seq + Arp as a discovery engine

Turn on the generative sequencer with moderate mutation (0.3–0.5), pick a scale, and let it run. Add the arpeggiator in Up mode at 1/16. The combination reveals how your sound behaves across pitch ranges and rhythmic contexts — a rapid audition tool.

10. Save early, save often

Presets include the generated audio, so you never lose a good generation. Export a preset the moment you find something promising. You can always refine the synth parameters later — but the generation itself is hard to reproduce without the exact seed and settings.

Part II

13. How Semantic Axes Work

T5ynth's semantic axes are directions in the 768-dimensional embedding space of the T5 text encoder. Here's how they are constructed:

Step 1: Define Poles

Each axis is defined by two single-word prompts representing opposing concepts — for example, "noise" and "music". These are not arbitrary labels; they were selected through systematic testing of which word-pairs produce the most audibly distinct results.

Step 2: Encode Poles + Neutral

Three embeddings are computed via the T5 encoder:

  • emb_pole_a — encoding of the negative pole (e.g., "noise")
  • emb_pole_b — encoding of the positive pole (e.g., "music")
  • emb_neutral — encoding of an empty prompt ("")

Step 3: Compute Direction

The axis direction is the difference between a pole and the neutral point:

  • Positive values: direction = emb_pole_b − emb_neutral
  • Negative values: direction = emb_pole_a − emb_neutral

This direction vector is then scaled by the slider value and added to the current manipulated embedding.

Step 4: Validation

Not all conceptual oppositions produce audible differences. The 8 axes included in T5ynth were validated by generating audio at each pole and measuring Mel-spectrogram cosine distance. Only axes with a distance above 0.70 at 1-second duration survived. PCA-based axes (derived from variance in the embedding space rather than semantic opposition) were tested but found to collapse at short durations — they produce near-identical audio regardless of slider position.

In the code, axis direction vectors are summed onto the embedding. But because the diffusion model processes the resulting embedding non-linearly, the audible effect is not simply additive — multiple axes are interrelated and can interfere with each other unpredictably. There is no orthogonalization, and axes may partially overlap in embedding space.

14. The DimExplorer In Depth

The DimExplorer (see section 3 for basic usage) operates on the mean-pooled T5 encoder output — the 768-dimensional vector obtained by averaging across the token sequence dimension, weighted by the attention mask. This is the same representation the diffusion model uses for conditioning.

Why Sorted by Magnitude?

Sorting the 768 bars by |emb_A[i] − emb_B[i]| is not just a display convenience — it surfaces the dimensions along which your two prompts are most semantically distinct. These high-magnitude dimensions are where the T5 encoder "disagrees" most about A and B. Editing these dimensions has the highest probability of producing an audible change; the low-magnitude bars on the right are dimensions where A and B are nearly identical and where edits tend to have little effect.

What Does a Dimension Represent?

Individual T5 dimensions do not correspond to human-interpretable audio features. The T5 model was trained on language tasks (not audio), and its internal structure encodes statistical patterns in text — not timbral qualities, pitch, or rhythm. The Stable Audio model learned to map these text patterns to sound, but the mapping is opaque and non-linear. A single dimension might contribute to brightness in one prompt context and to rhythmic density in another. This is why the DimExplorer is a research tool: it rewards experimentation, not prediction.

Technical Details

  • Offsets are applied as manipulated[:, :, idx] += offset_value — broadcast across all tokens in the sequence
  • Only non-zero offsets are transmitted to the inference engine (sparse representation)
  • Offsets persist across generations until explicitly reset
  • The edit history (undo/redo) stores complete offset state snapshots

15. Principles of the Generative Sequencer

The generative sequencer combines three layers of musical algorithm:

Layer 1: Euclidean Rhythm

The rhythmic backbone uses Bjorklund's algorithm (Toussaint, 2005) to distribute N pulses across M steps as evenly as possible. This is the same mathematical principle behind many traditional rhythmic patterns worldwide — it produces patterns that feel natural because they maximize spacing regularity.

Example: 5 pulses in 8 steps → [x . x . x . x x] — a pattern equivalent to Cuban tresillo.

The Rotation parameter shifts this pattern circularly, changing which beat feels like "one."

Layer 2: Stride-Based Melody

Pulses are assigned pitches using a stride permutation through the selected scale. The stride value (max(1, steps − pulses)) determines how many scale degrees to skip between successive notes, chosen to be coprime with the total number of degrees so the melody visits as many pitches as possible before repeating.

The traversal follows a triangle wave (ascending then descending), creating bidirectional melodic motion rather than monotonic climbing. This produces melodies that feel more natural and song-like.

Velocity is derived from the rhythmic gaps — notes after longer rests receive higher velocity, creating natural accents at the start of rhythmic phrases.

Layer 3: Turing Machine Mutation

Each time the pattern completes a cycle, Mutation determines how many notes change. At mutation rate 0.8 with 16 pulses, roughly 8 notes shift by 1–4 scale degrees. The result: the pattern stays recognizable but continuously evolves — like a melodic idea being rethought in real time.

The mutation rate itself breathes sinusoidally over a 16-cycle window, oscillating between 50% and 100% of its set value. This creates alternating periods of stability and flux without any user input.

Parameter Drift

When the Fix toggles are unlocked, the Euclidean parameters themselves drift over 8-cycle windows:

  • Rotation shifts circularly (timing determined by a secondary Euclidean distribution)
  • Pulses increase or decrease over time, changing pattern density
  • Steps expand or contract while preserving the pulse-to-step ratio

All drift is bounded and reversible — parameters oscillate rather than running away. The combination of melodic mutation, mutation breathing, and parameter drift produces patterns that can evolve for minutes without becoming monotonous or losing coherence.

16. The Inference Pipeline

Understanding the inference pipeline helps explain why certain controls interact the way they do.

JUCE (GUI) Python (Inference) │ │ ├─ Prompt A text ──────────────────── T5 Tokenizer + Encoder ─── emb_a [1, seq, 768] ├─ Prompt B text ──────────────────── T5 Tokenizer + Encoder ─── emb_b [1, seq, 768] │ │ ├─ Alpha ─────────────────────────── Interpolate(emb_a, emb_b, alpha) │ │ ├─ Dimension Offsets ─────────────── Add per-dim offsets to embedding │ │ ├─ Semantic Axes ─────────────────── Add scaled axis directions │ │ ├─ Magnitude ─────────────────────── Scale embedding vector length ├─ Noise ─────────────────────────── Add Gaussian noise (PCG64 RNG) │ │ ├─ Steps, CFG, Duration, Seed ────── Diffusion sampling (BrownianTree) │ │ │ VAE Decoder → audio [ch, samples] @ 44.1 kHz │ │ │ HF Boost + Rumble Filter │ │ │◄──── Pipe (binary) ────────────── PCM float32 + embeddings [768] × 2 │ Load into Engine (Sampler or Wavetable) │ Per-Voice DSP: Envelope → Filter → Modulation │ Master FX: Delay → Reverb → Limiter → Output

Key observations from this flow:

  • Prompts, Alpha, Axes, Magnitude, Noise, and DimExplorer offsets all operate before diffusion — they shape the embedding that guides generation.
  • Filter, envelopes, LFOs, effects operate after generation — they are traditional synth processing on the generated audio.
  • Drift LFOs bridge both worlds — they can modulate pre-generation parameters (Alpha, Axes) and post-generation parameters (Filter, Scan).
  • The Python inference subprocess stays alive between generations — no startup overhead.
  • Determinism is guaranteed across platforms (same seed + settings = identical audio on Mac, Linux, GPU, or CPU) via a patched numpy PCG64 random number generator.
